[VTB Благотворительность]

    Это платформа для сбора и учета благотворительных пожертвований. Приложение позволяет пользователям отправлять средства в различные категории (ДЕТИ, ЖИВОТНЫЕ, БОЛЬНЫЕ) и отслеживать общую статистику сборов.

Функционал

    Система пожертвований: Обработка и запись новых пожертвований в базу данных.

    Перенаправление: Динамическое перенаправление пользователя на целевую страницу в зависимости от выбранной категории.

    Статистика в реальном времени: Отображение агрегированных данных о суммах и количестве пожертвований (Всего, Детям, Другое).

    Административная панель: Управление записями о пожертвованиях через встроенный интерфейс Django Admin.
Технологический стек

Предварительные требования

Для локального запуска проекта вам потребуется установленный [Python 3.x] и Git.

    Python (рекомендуется): 3.10+

    Git: любая версия

Локальная установка

Следуйте этим инструкциям, чтобы запустить копию проекта на вашем локальном компьютере для разработки и тестирования.

1. Клонирование репозитория

Bash

git clone [vtbblag.github.io]
cd [vtb]

2. Создание виртуального окружения

Настоятельно рекомендуется использовать виртуальное окружение для управления зависимостями.
Bash

# Для Linux/macOS
python3 -m venv venv
source venv/bin/activate

# Для Windows
python -m venv venv
venv\Scripts\activate

3. Установка зависимостей

Установите все необходимые библиотеки из файла requirements.txt:
Bash

pip install -r requirements.txt

4. Настройка переменных окружения

Создайте файл .env (или используйте переменные окружения вашей системы) для хранения секретных данных.

Пример .env:

SECRET_KEY='[django-insecure-2knd2kbvvbzc@(_x8_gy5!qf1)0pel!(czkf_=060n&zn16fi$]'
DEBUG=True

Настройка базы данных

1. Применение миграций

Выполните миграции, чтобы создать таблицы в базе данных:
Bash

python manage.py migrate

2. Создание суперпользователя (Администратора)

Для доступа к Django Admin:
Bash

python manage.py createsuperuser
# Следуйте инструкциям (введите имя и пароль)

3. Сбор статических файлов

Обязательный шаг для корректного отображения CSS/JS в production, но полезен и в разработке:
Bash

python manage.py collectstatic

Использование

Запустите локальный сервер Django:
Bash

python manage.py runserver

Приложение будет доступно по адресу: http://127.0.0.1:8000/

Административная панель: http://127.0.0.1:8000/admin/

Пользователь: admin
Пароль: 1stta

Основные требования для Production

    Убедиться, что DEBUG = False в settings.py.

    Настроить ALLOWED_HOSTS для вашего домена.

    Настроить базу данных (рекомендуется SQLite).

    Настроить статический и медиа-файлы (обычно через AWS S3 или аналогичный сервис).
